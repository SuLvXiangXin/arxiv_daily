Warm-Starting Optimization-Based Motion Planning for Robotic Manipulators via Point Cloud-Conditioned Flow Matching

arXiv: 2510.03460 作者: Xiao Liang Team 日期: 2025-10-03

📝 详细解读

研究背景与动机

机器人运动规划主要分为基于采样的方法（如RRT、PRM）和基于优化的方法。基于采样的方法概率完备，但在高维构型空间中面临维度诅咒，且生成的路径通常需要后处理平滑，在复杂环境中效率较低。基于优化的方法能直接生成满足多种约束的平滑轨迹，但其目标函数高度非凸，对初始化敏感，容易陷入局部最小值。常见的线性插值初始化策略在失败时，需借助耗时更长的采样方法获取更好的初始解。

本文针对优化方法对初始化敏感这一具体痛点，提出利用学习模型为轨迹优化提供高质量“热启动”的新视角。现有学习型热启动方法通常依赖已知的障碍物位置与几何信息，或采用确定性模型仅生成单一初始解，限制了其在动态、部分可观测环境中的适用性和鲁棒性。扩散模型虽能基于感知输入生成解，但多步去噪过程导致推理速度较慢。本文核心思路是：利用基于单视点云条件的流匹配模型，快速生成多个多样化、近最优的轨迹种子，以热启动GPU加速的轨迹优化器，从而在无需先验障碍物知识的情况下，实现快速、鲁棒的机器人运动规划。

方法详解

本文提出一个学习增强的轨迹优化框架，核心是流匹配基础的随机神经初始器，后接并行轨迹优化器。

整体流程：给定起始构型、目标构型及单视深度图像，首先从深度图像生成点云。点云与任务信息（起始、目标构型）一同输入到流匹配初始器中，该初始器批量生成多个轨迹种子。随后，从点云估计障碍物网格，并与所有轨迹种子一起送入GPU加速优化器cuRobo进行批量并行优化，最终输出多个优化后的轨迹供选择。

图2：学习增强的轨迹优化框架架构。左侧为基于流匹配的神经初始器，它以起始构型、目标构型、点云和时间步t为条件，通过SE-Transformer块估计速度场，从而生成轨迹种子。右侧为批量轨迹优化器（cuRobo），利用从点云估计的障碍物网格进行碰撞检查，对多个种子进行并行优化。

核心模块1: 流匹配基础的随机神经初始器。该方法采用流匹配学习从简单分布（标准正态分布）到复杂轨迹数据分布的映射。定义速度场 ( u_{\phi}(\mathbf{x}_t, t) ) 由神经网络参数化。训练时采用直线路径假设，即中间样本 ( \mathbf{x}_t = (1-t)\mathbf{x}0 + t\mathbf{x}1 )，损失函数为 ( \mathcal{L}{\text{FM}}(\phi) = \mathbb{E} [| u{\phi}(\mathbf{x}_t, t) - (\mathbf{x}_1 - \mathbf{x}_0) |^2 ] )。训练后，从正态分布采样 ( \mathbf{x}_0 )，通过积分学习到的速度场即可生成目标分布样本（即轨迹种子）。

神经初始器网络架构：输入是轨迹表示 ( \tau \in \mathbb{R}^{K \times J} )（K=128，J=6）、起始构型 ( \theta_0 )、目标构型 ( \theta_g ) 以及带标签（障碍物/机器人）的点云 ( P \in \mathbb{R}^{N \times 4} )。轨迹经过编码层和可学习位置编码后，送入一系列SE-Transformer块。点云特征通过PointNet++ Set Abstraction模块提取，起始和目标构型分别用线性层编码，时间步t用MLP编码。所有条件嵌入拼接后，通过自适应层归一化（AdaLN）模块调节SE-Transformer各层的潜变量，最终由线性层输出估计的速度场。该设计支持批量生成多个多样化种子。

核心模块2: 学习增强的轨迹优化框架。生成的多个轨迹种子被送入cuRobo优化器进行批量并行优化。cuRobo先进行几轮基于粒子的优化引导种子进入有希望的区域，然后应用L-BFGS快速收敛。关键的是，优化所需的障碍物信息并非预先给定，而是从同一单视点云估计出的网格，因此整个框架仅需深度观测，无需环境先验知识。

创新点：1) 快速多样化种子生成：利用流匹配模型，仅需1-2步推理即可批量生成多个近最优轨迹种子，相比扩散模型推理更快。2) 以点云为条件的感知集成：直接以原始点云为条件，使方法适用于部分可观测的动态环境，无需障碍物先验。3) 随机初始器与批量优化的协同：随机初始器提供多样性，cuRobo进行批量并行优化，共同提升成功率和在动态环境中重规划的效率。

实验与结果

实验设置：使用UR5e六自由度机械臂模型，在模拟的杂乱工作空间中进行评估。数据集基于SIMPNet环境生成，包含7个不同障碍物布置的场景，共4416个具有可行解的运动规划问题。轨迹长度K=128。保留一个场景（Env 7）作为未见环境测试集。

对比基线：

  1. Linear Interpolation：线性插值作为单一种子。
  2. Transformer：基于Transformer的确定性神经初始器（编码器-解码器架构）。
  3. Linear Interpolation-batch：将线性插值结果复制10份进行批量优化。
  4. Transformer-batch：将Transformer初始器结果复制10份进行批量优化。
  5. **FM (Ours)**：本文提出的流匹配初始器，生成10个不同种子进行批量优化。评估了一步推理和两步推理两种模式。

评估指标：成功率（基于真实障碍物碰撞检查的可行解百分比）和规划时间。

图3：方法生成的轨迹定性示例。左图展示了从单视点云（粉色）生成的多条优化后轨迹（不同颜色），右图对比了线性插值初始化（红色）失败，而本文方法初始化（绿色）成功的情况。

关键定量结果：

  1. 与基础基线对比（表1）：在已见环境（Env 1-6）中，仅使用本文初始器（FM Init.）的成功率为43.7%，显著高于线性插值（33.9%）和Transformer（43.7%）。经过100次优化迭代后，FM初始器结合优化的成功率高达93.5%（表2，两步推理），远优于Transformer-batch（90.4%）和Linear Interpolation-batch（67.5%）。
  2. 与学习基线对比（表2）：在大多数环境中，无论是初始器单独使用还是结合优化，本文方法（FM）的成功率均高于或与Transformer-batch持平。特别是在优化早期（5次迭代），FM的优势更为明显，表明其提供的种子质量更高，能加速优化收敛。
  3. 泛化能力：在未见环境（Env 7）中，FM初始器（两步推理）经过100次优化后成功率高达93.2%，与已见环境性能相当，展现了强大的泛化能力。
  4. 推理步骤消融：两步推理的FM初始器性能普遍优于一步推理，但一步推理已能提供质量很高的初始解（单独成功率41.3%，优化后91.7%），体现了流匹配的高效性。

图4：不同方法的规划时间对比。本文方法（FM）在提供更高成功率的同时，其总规划时间与Transformer-batch相当，且显著快于需要多步rollout的模仿学习方法（如SIMPNet）。

消融实验总结：两步推理相比一步推理能进一步提升初始解质量和最终成功率，但一步推理已具备实用价值。批量生成多样化种子（FM）相比复制确定性种子进行批量优化（Transformer-batch）能带来显著的成功率提升，证明了随机初始器提供多样性对于避免局部最优的重要性。

总结与启发

核心贡献：

  1. 提出了一个基于流匹配的随机神经初始器，能够以极少的推理步骤（1-2步）批量生成多样化、近最优的轨迹种子，为轨迹优化提供了高效、鲁棒的热启动。
  2. 实现了以单视点云为条件的运动规划，使整个框架无需环境障碍物的先验位置与几何知识，可直接从深度相机观测生成可行轨迹，适用于部分可观测的动态环境。
  3. 通过模拟实验验证了该框架能显著提升轨迹优化的成功率和收敛速度，并展现出对未见环境的良好泛化能力。

局限性：论文提到，障碍物网格的估计质量依赖于输入点云的分割效果。此外，所有实验均在模拟环境中进行，尚未在真实机器人系统上验证。

后续研究启示：本文展示了流匹配生成模型在机器人运动规划中兼具速度与质量的潜力。未来工作可探索更鲁棒的点云处理与网格估计方法，并将框架部署到真实机器人平台，在动态人机协作场景中测试其在线重规划性能。此外，如何将更高阶的动态约束更有效地集成到学习初始器中，也是一个值得探索的方向。

💡 一句话总结

本文提出一种基于学习的机器人运动规划方法，用于解决动态环境中优化轨迹生成易陷入局部最优、依赖初始解的问题。核心方法是利用单视点云条件流匹配模型，直接从深度相机输入学习近似最优的初始化轨迹，无需障碍物先验知识。在UR5e机械臂的仿真实验中，该方法自身成功率高，相比基准方法显著提升了轨迹优化的成功率，减少了优化迭代次数，并对未见环境表现出强泛化能力。
