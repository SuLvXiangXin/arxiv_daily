SADCHER: Scheduling using Attention-based Dynamic Coalitions of Heterogeneous Robots in Real-Time

arXiv: 2510.14851 作者: Javier Alonso-Mora Team 日期: 2025-10-16

📝 详细解读

研究背景与动机

异构多机器人系统在复杂现实场景中具有广泛应用，如灾难响应、自主建造等。高效的多机器人任务分配（MRTA）算法至关重要。根据现有分类法，本文研究的是单任务机器人、多机器人任务、时间扩展分配且具有交叉依赖（ST-MR-TA-XD）的设置，其挑战包括处理动态联盟形成、任务优先级约束以及实时适应环境变化。现有方法中，混合整数线性规划（MILP）能提供精确解但计算复杂度过高，无法实时应用；基于学习的方-法，特别是模仿学习（IL），通过从专家数据中学习以实现快速推理，但先前工作（如HeteroMRTA）在网络架构中忽略了机器人/任务的位置和持续时间信息，这限制了其时空推理能力和对未见时空分布的泛化能力。本文针对这一具体痛点，提出通过显式建模机器人/任务位置、任务持续时间和机器人剩余处理时间，以实现高级时空推理和泛化。核心思路是结合图注意力网络和Transformer来预测机器人-任务分配奖励，并通过松弛的二分图匹配生成高质量、可行的调度方案。

方法详解

SADCHER框架包含一个基于注意力机制的神经网络和一个松弛的二分图匹配器。在运行时，该方法在决策步骤（如机器人完成任务或新任务发布时）异步地重新计算分配。

图2：SADCHER架构总览。机器人和任务图分别经过图注意力（GAT）和Transformer编码器处理，并与计算出的距离特征拼接。奖励矩阵由空闲MLP和任务奖励MLP估计，最终任务分配通过松弛二分图匹配提取。

整体流程：输入是机器人状态图 𝒢^r 和任务依赖图 𝒢^t。网络首先为每个机器人和任务生成嵌入，然后通过GAT编码器捕获局部图结构信息，再通过Transformer编码器建立全局上下文。同时，一个距离多层感知机（MLP）处理机器人-任务对的归一化欧氏距离。接着，将机器人和任务的GAT输出、Transformer输出以及距离特征拼接，形成每个机器人-任务对的信息丰富特征向量。该特征向量被送入两个独立的MLP头：一个预测将该机器人分配给该任务的奖励（R_i,j_task），另一个输出一个鼓励空闲的信号。对所有任务的空闲信号求和得到每个机器人的空闲奖励（R_i_IDLE）。最终，所有机器人-任务对奖励与空闲奖励共同组成奖励矩阵 R。R 被送入一个考虑技能覆盖、机器人可用性和任务就绪状态的松弛二分图匹配问题，求解得到当前决策步的最优分配矩阵 A*。

核心模块与技术细节：

  1. 图注意力（GAT）编码器：分别处理机器人图和任务图。机器人GAT假设全连接图，实现机器人间的全注意力。任务GAT则利用编码了优先级约束的邻接矩阵，使任务节点能关注其前驱或后继任务，从而理解任务时序逻辑。GAT通过计算节点间的注意力权重（公式2）来聚合邻居信息，输出捕获局部关系的表征 h^GAT_R 和 h^GAT_T。
  2. Transformer编码器：在GAT编码器之后，独立的Transformer编码器对机器人和任务表征进行全局上下文建模。通过多头自注意力机制（公式4,5），使每个机器人/任务能关注所有其他机器人/任务，输出 h^T_R 和 h^T_T。
  3. 距离特征与奖励预测：显式计算机器人i与任务j的归一化距离，并通过一个MLP_D映射为距离特征 d_i,j（公式6）。然后将机器人i的GAT和Transformer输出、任务j的GAT和Transformer输出以及 d_i,j 拼接为特征向量 f_i,j（公式7）。f_i,j 分别通过MLP_R和MLP_I预测任务奖励和空闲贡献（公式8）。
  4. 松弛二分图匹配：分配问题被形式化为一个优化问题（公式9），在满足约束（10-12）下最大化所选边的奖励总和。约束包括：每个机器人最多分配一个任务（或空闲）、分配给每个任务的机器人联盟其技能总和必须覆盖任务所需技能、只匹配空闲机器人和就绪任务。匹配后，会移除联盟中对覆盖任务技能无唯一贡献的冗余机器人。
  5. 预移动策略：如果机器人被分配了空闲任务，它会向其奖励最高的任务方向移动，但不正式分配。这能在下次重规划时可能减少任务启动延迟。

创新点：与之前类似架构的IL方法（如[30]）相比，SADCHER的关键创新在于：1) 引入了距离MLP，显式提供空间邻近信息；2) 网络输入显式包含了机器人位置、任务位置、任务持续时间和机器人剩余任务时间，支持时空推理；3) 使用了改进的损失函数（公式14），同时鼓励准确预测可行奖励并抑制不可行奖励的高值；4) 引入了空闲任务和预移动策略，以通过延迟分配实现更好的长期调度。

实验与结果

实验设置：使用自行生成的包含250,000个最优解的数据集进行训练和评估。测试在500个随机生成的、未见过的实例上进行（默认规模：8任务、3机器人、3个优先级约束）。实验平台为消费级硬件（AMD Ryzen 7 4800H CPU, NVIDIA GeForce GTX 1650 GPU）。

对比方法：

  1. MILP：带优先级约束的最优求解器，作为性能上限。
  2. HeteroMRTA：基于强化学习的方法的确定性变体。
  3. S-HeteroMRTA：HeteroMRTA的采样变体（10次运行取最佳）。
  4. Greedy：基于启发式（优先满足最多技能需求，按旅行时间断联）的贪婪算法。
  5. Sadcher：本文提出的确定性方法。
  6. S-Sadcher：Sadcher的采样变体（对预测奖励进行采样，10次运行取最佳）。

图3：在500个未见实例上关于完工时间（左）和计算时间（右）的对比。S-Sadcher和Sadcher在完工时间上最接近最优解（MILP）。Sadcher的每次决策时间（~22 ms）满足实时性要求，远快于MILP的完整求解时间（平均76秒）。

图4：HeteroMRTA与Sadcher（左）、S-HeteroMRTA与S-Sadcher（右）的完工时间成对比较图。大多数点位于对角线以下，表明Sadcher在绝大多数实例上表现更优。

关键实验结果：

  • 完工时间（Makespan）：在训练域规模问题上，S-Sadcher（相对最优解平均差距3.8%）和Sadcher（差距6.8%）是性能最好的非最优算法，显著优于S-HeteroMRTA（差距10.8%）、HeteroMRTA（差距21.5%）和Greedy（差距20.4%）。如图4所示，Sadcher在80.6%的实例上击败了HeteroMRTA。
  • 计算时间：对于实时决策至关重要的每次决策时间，Greedy最快（0.080 ms），HeteroMRTA次之（9.1 ms），Sadcher为22 ms，仍适合实时操作。MILP的完整求解时间平均为76秒，无法实时使用。
  • 对优先级约束的理解：Sadcher通过优先分配前驱任务来展示其对任务依赖关系的理解，平均分配就绪前驱任务的频率比基线方法高约1.7倍。
  • 域外泛化与消融实验：

图5：缩放实验（左上、左下）和消融实验（右上、右下）结果。左图显示，当机器人数量（3, 5, 20）和任务数量（8, 16, 32）增加时，Sadcher相对于基线的完工时间优势普遍保持或扩大。右图的消融实验表明，距离特征（-Dist）和空闲任务（-Idle）都对性能有重要贡献，移除任一都会导致性能下降。

  • 缩放性：如图5左列所示，当团队规模（最多20机器人）和任务集（最多32任务）扩大到远超训练数据（3机器人，8任务）时，Sadcher相比HeteroMRTA和Greedy仍能保持或扩大其完工时间优势，展示了良好的泛化能力。
  • 消融实验：图5右列展示了在训练域规模上的消融结果。移除距离特征（-Dist）导致性能下降约5.5%，移除空闲任务（-Idle）导致性能下降约7.5%，而两者都移除（-Both）则下降约13.6%，验证了这两个设计组件的有效性。

总结与启发

核心贡献：

  1. 提出了SADCHER框架，一个结合了图注意力、Transformer和松弛二分图匹配的实时异构多机器人任务分配方法，能够处理动态联盟形成和任务优先级约束，并显式建模时空信息以实现高级推理和泛化。
  2. 创建并公开了一个包含250,000个最优解的大规模数据集，可用于模仿学习训练或作为基准。
  3. 通过大量实验证明，SADCHER在性能和计算效率上优于现有的学习和启发式基线方法，并能有效地泛化到远超训练规模的场景中。

局限性：论文提到，训练数据的生成依赖于解决小规模实例的MILP，计算成本较高。此外，网络架构（如层数、头数）可能并非最优，存在进一步调整的空间。

启示：公开的数据集为后续基于学习的MRTA研究提供了宝贵资源。SADCHER的框架设计，特别是显式融入时空信息和距离特征以增强泛化能力的思路，对解决其他需要复杂时空推理的调度或规划问题具有借鉴意义。将学习-based的奖励预测与保证可行性的组合优化步骤相结合，是平衡学习泛化能力与解可靠性的有效途径。

💡 一句话总结

本文提出SADCHER框架，旨在解决异构多机器人团队在动态环境中，考虑任务优先级约束与动态联盟形成的实时任务分配问题。其核心技术是结合图注意力网络与Transformer的模仿学习方法，预测机器人与任务间的分配奖励，并通过松弛二分匹配生成可行调度。实验表明，该方法在随机未见问题上优于其他学习与启发式基线，计算时间满足实时要求，且通过小规模最优解训练后，能泛化至更大规模的团队与任务集。
