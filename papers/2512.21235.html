RoboCade: Gamifying Robot Data Collection

arXiv: 2512.21235 作者: Dorsa Sadigh Team 日期: 2025-12-26

📝 详细解读

研究背景与动机

模仿学习已成为训练机器人策略的主流方法，但其依赖于大量人类演示数据。当前数据收集过程存在两大瓶颈：一是需要接触昂贵的实体机器人硬件，限制了参与者范围；二是即使通过远程遥操作平台降低了物理接触门槛，数据收集过程本身枯燥乏味，缺乏用户参与激励，导致数据规模受限。本文针对数据收集的可扩展性挑战，提出了游戏化的新视角，旨在通过设计一个既有趣又易于访问的游戏化平台，吸引更广泛的用户参与，从而规模化地收集对下游策略训练有益的演示数据。本文核心思路是：将远程遥操作重构为游戏化体验，通过在系统界面和任务设计中嵌入游戏化元素，使数据收集过程变得引人入胜，并证明由此收集的数据能有效提升下游机器人策略的性能。

方法详解

RoboCade 的整体框架是一个基于 Web 的游戏化远程遥操作平台，其输入是用户通过低成本、可 3D 打印的 GELLO 控制器发出的指令，输出是真实机器人执行的动作序列（即演示数据），同时为用户提供沉浸式的游戏化交互界面。其核心创新在于将游戏化原则系统性地融入系统设计与任务设计两大模块。

核心模块一：游戏化系统设计
该模块旨在将标准的遥操作界面转化为吸引人的游戏体验，遵循四大设计原则：

  1. **反馈与奖励 (SD1)**：提供即时、多模态反馈以强化用户行为。例如，视觉上高亮被抓取的物体，任务成功时播放庆祝音效和显示虚拟彩花。
  2. **挑战与目标提示 (SD2)**：为每个任务设定具有挑战性的时间限制，并通过进度条可视化。任务目标以主题化的方式呈现（如将待扫描物品清单设计为购物小票）。
  3. **身份与进度 (SD3)**：用户拥有包含用户名和头像的持久个人资料。参与游戏积累点数并可解锁徽章，形成轻量级的进度系统。
  4. **社交互动 (SD4)**：设立公开排行榜，按累计游戏点数排名。每个游戏会话生成唯一可分享的URL，允许他人实时观看，引入竞争与观赏元素。

核心模块二：游戏化任务设计
此模块旨在设计既有趣又能产生有用数据的“支持任务”，作为下游“目标任务”的数据补充。设计遵循四大原则，实现从目标任务 τ 到游戏化支持任务 τ' 的转换：

  1. **叙事 (TD1)**：为功能性的动作序列赋予故事主题（如“帮助动物回家”），通过文本和视觉提示增强沉浸感。
  2. **目标多样性 (TD2)**：在保持底层操作技能不变的前提下，变化每轮任务的具体目标（如更换物体、改变空间布局），增加新鲜感。
  3. **挑战性 (TD3)**：校准任务难度，使其对新手而言非平凡但通过练习可达成，以维持“心流”体验。
  4. **技能重叠 (TD4)**：确保支持任务至少包含一项与目标任务共享的高层操作技能（如抓取相似几何形状的物体、插入动作）。

系统实现细节
平台基于 Franka FR3 机械臂搭建，使用 GELLO 控制器进行关节阻抗控制，实现了直观且低成本的遥操作。后端使用 ZeroMQ 进行消息传递，前端使用 Next.js 和 Three.js 构建 Web 界面并实现 3D 渲染，通过 WebSockets 实现前后端实时通信，确保了远程操作的流畅性和游戏化元素的低延迟呈现。

实验与结果

实验使用了三个操作任务对（目标任务及对应的游戏化支持任务）作为 benchmark，在真实机器人上评估。Baseline 方法是仅使用目标任务数据训练策略（Target Only），对比方法是使用目标任务数据与游戏化支持任务数据协同训练（Co-train）。策略模型采用 Diffusion Policy。此外，还进行了用户研究（N=18）比较游戏化平台与标准非游戏化平台的用户体验。

图4：协同训练策略性能对比。柱状图展示了三个目标任务在仅用目标数据训练和用（目标+游戏化支持）数据协同训练下的分阶段成功率。协同训练显著提升了所有目标任务在分布内（In-Dist.）的性能（ArrangeDesk: +16%, ScanBottle: >50%, PackBox: +16%）。对于 ScanBottle 和 PackBox，在分布外（Out-of-Dist.）初始配置下的测试显示，协同训练还能提升泛化能力（提升约20%）。

图5：视觉-语言-动作模型协同微调结果。展示了在预训练的 π 0.5 模型基础上，仅用目标任务数据微调与用（目标+游戏化支持）任务数据协同微调的性能对比。协同微调在分布内条件下保持或略超性能，同时在分布外条件下显著提升了泛化能力（ScanBottle 的 Pick Bottle 阶段从40%提升至88%）。

关键实验结果总结：

  1. 策略性能提升：在固定训练步数下，协同训练相比仅使用目标数据，将三个目标任务的最终成功率分别提升了 16%（PackBox）、超过 50%（ScanBottle）和 16%（ArrangeDesk，从12%到28%）。定性观察显示，协同训练后的策略抓取更精准、动作质量更高。
  2. 泛化能力增强：对于 ScanBottle 和 PackBox，协同训练使策略在分布外初始配置下的性能提升了约20%，证明了游戏化数据有助于提升泛化性。
  3. 大模型微调受益：对预训练的视觉-语言-动作模型进行协同微调，也能在保持分布内性能的同时，显著提升其在分布外条件下的表现（例如，ScanBottle的抓取成功率从40%提升至88%）。
  4. 用户体验显著改善：用户研究表明，与标准非游戏化平台相比，用户认为 RoboCade 平台更直观（+27%）、更令人享受（+24%）、更激励人参与（+24%）。

总结与启发

核心贡献：

  1. 提出了 RoboCade，一个集成了系统化游戏化设计的低成本、远程机器人数据收集平台，旨在通过提升趣味性和可访问性来规模化数据收集。
  2. 提出了一套用于设计游戏化机器人操作任务的原则，确保这些任务既能吸引用户，又能产生与下游目标任务技能重叠的有用演示数据。
  3. 通过实验证明，利用该平台收集的游戏化数据与目标任务数据协同训练或微调策略，能显著提升策略在目标任务上的成功率和泛化能力，并验证了平台在吸引用户方面的优越性。

局限性：
论文提到的局限性包括：目前探索的任务范围有限（三种操作任务）；游戏化任务的设计仍需人工根据设计原则进行，尚未自动化；研究主要关注模仿学习范式。

后续研究启示：

  1. 自动化游戏化设计：未来工作可以探索如何自动生成或适配游戏化任务，以进一步降低应用门槛并扩展任务类型。
  2. 探索其他学习范式：可将游戏化数据收集的理念应用于强化学习、课程学习等其他机器人学习范式。
  3. 长期参与机制：研究如何通过更丰富的游戏机制（如更复杂的叙事、社交功能）维持用户的长期参与，实现持续、大规模的数据收集。
  4. 技能组合与泛化：探索如何通过设计包含多种基础技能组合的游戏化任务，来训练更具泛化能力的机器人策略。

💡 一句话总结

本文针对模仿学习数据收集成本高、过程枯燥、可扩展性差的核心问题，提出RoboCade游戏化远程遥操作平台。关键技术包括在系统界面集成视觉反馈、进度条、排行榜等游戏化元素，并设计任务结构使其与下游目标任务重叠。实验表明，利用该平台收集的数据进行共同训练，可将非游戏化目标任务的成功率提升16%-56%；用户研究证实，游戏化设计使新手用户的愉悦度显著提高24%。
